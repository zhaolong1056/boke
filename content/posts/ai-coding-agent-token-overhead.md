## 前言

用 AI 编码 Agent 写代码，爽是真爽，但你有没有想过 —— **每次对话还没开始，你的 Token 就已经被烧掉了一大截？**

这就像去自助餐厅，还没坐下就得先交一笔"座位费"。这笔"座位费"就是**前置 Token（Upfront Tokens）**：系统提示词（System Prompt）、工具定义（Tool Definitions）、上下文注入（Context Injection）等固定开销。无论你问什么，只要你打开对话窗口，这些 Token 就已经被计入消耗。

本文将对 **7 款主流 AI 编码 Agent / 工具** 进行前置 Token 横评，帮你在选型时把这笔隐形账算清楚。

## 什么是"前置Token"？

在细说之前，先统一口径。这里的 **前置Token** 指：

1. **系统提示词（System Prompt）** —— 定义 AI 角色、行为规则、输出格式的固定文本
2. **工具 / 函数定义（Tool Definitions）** —— Agent 可调用的所有工具的 JSON Schema
3. **上下文注入（Context Injection）** —— 编辑器状态、光标位置、诊断信息、快捷键等环境数据
4. **Memory / 记忆注入** —— 持久化的跨会话记忆内容

这些 Token **在用户输入任何内容之前就已经被消耗**，是真正的"固定成本"。

## 各工具前置Token逐项分析

### 1️⃣ Claude Code — Token大户

Claude Code 是 Anthropic 推出的终端 AI 编码 Agent，功能强大，但代价也不小。

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~3,800 |
| 工具定义（约 30+ 工具） | ~2,500 |
| 角色/行为指令 | ~1,200 |
| **合计前置Token** | **~7,200** |

Claude Code 有非常详细的系统提示词，定义了其作为编码 Agent 的完整行为准则，包括代码编辑的规范、文件操作的安全约束、shell 命令执行的权限模型等。加上近 30 个原生工具的定义（读文件、写文件、搜索、执行命令、项目管理等），每个工具的 JSON Schema 都要传输，堆叠起来 Token 量非常可观。

**上下文窗口**: 200K tokens（Claude 4 Sonnet）
**定价**: $20/月 Pro 订阅，或按 API Token 计费

> 💡 一个来自 Claude Code 用户的实测：对话刚开始、还没说一句话，日志里就已经显示消耗了 7,200 个 prompt tokens。

---

### 2️⃣ OpenCode — 轻量级开源方案

OpenCode 是由 Steve Yegge（前 Amazon 工程师、Google 技术传教士）发起，现为开源社区的编码 Agent 项目。

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~1,500 |
| 工具定义（约 10-15 个工具） | ~1,200 |
| 角色/行为指令 | ~800 |
| **合计前置Token** | **~3,500** |

相比 Claude Code，OpenCode 的设计哲学就是"够用就好"。工具数量少、功能定义精简、没有冗余的角色设定词。这让它的前置 Token 开销只有 Claude Code 的一半左右。

**上下文窗口**: 随底层模型而定（128K-200K）
**定价**: 完全免费开源，自备 API Key
**定价成本**: 按 API Token 计费，取决于你挂载的模型

> 💡 开源方案的最大优势：你可以自己改系统提示词，去掉不需要的工具，进一步压减前置 Token。

---

### 3️⃣ Cursor — IDE 的隐形开销

Cursor 是目前最火的 AI 原生编辑器之一，它的前置 Token 构成比较复杂，因为要和 IDE 状态深度绑定。

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~2,000 |
| 工具定义（Command + 上下文指令） | ~1,800 |
| IDE 状态注入（光标、文件树、诊断等） | ~1,200-2,000 |
| **合计前置Token** | **~5,000-5,800** |

Cursor 比较特殊：它的前置 Token 中有相当大的比例来自 **IDE 上下文注入**。每次对话时，Cursor 会将当前打开的文件、光标位置、编辑器诊断信息等打包注入。虽然这让 AI 更了解你的工作上下文，但每打开一个新对话，这些数据都要重新传输一遍。

注意：Cursor 的 Tab（自动补全）模式非常轻量，几乎没有前置开销，但 Chat 模式的成本就跟上面差不多了。

**上下文窗口**: 模型依赖（GPT-4o 128K / Claude 200K）
**定价**: $20/月 Pro，$40/月 Business

---

### 4️⃣ GitHub Copilot — 两副面孔

GitHub Copilot 有两种工作模式，前置 Token 开销天差地别。

**补全模式（Completions）：**

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~200 |
| 代码上下文切片 | ~300-500 |
| **合计前置Token** | **~500-700** |

**Chat 模式（Copilot Chat）：**

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~1,500 |
| 工具/命令定义（/fix, /explain 等） | ~800 |
| IDE 上下文注入 | ~600-1,000 |
| **合计前置Token** | **~2,900-3,300** |

Copilot 的补全模式之所以非常轻量，是因为它**不需要完整的 Agent 循环**。它本质上是一个"预测下一个字符"的模型在干活，不走工具调用，不需要函数定义，也不需要复杂的行为约束。这是目前所有工具里**前置 Token 最少**的工作模式。

但 Chat 模式就要重得多，毕竟要支持 /fix、/tests、/explain 等 slash command，每个命令背后都有对应的系统指令。

**上下文窗口**: 模型依赖
**定价**: $10/月 Individual，$19/月 Business，$39/月 Enterprise

---

### 5️⃣ OpenAI Codex CLI — 后来者的平衡之道

OpenAI 在 2025 年底正式推出 Codex CLI，作为与 Claude Code 直接竞争的产品。

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~2,200 |
| 工具定义（约 12-15 个工具） | ~1,800 |
| 沙箱/安全约束指令 | ~800 |
| **合计前置Token** | **~4,800** |

Codex CLI 的前置 Token 处在中间位置，比 Claude Code 轻不少，但比 OpenCode 重。它的工具定义比较精简（不需要手写大量编辑器命令），沙箱安全指令也写得比较克制。整体而言，OpenAI 在这点上做了较好的平衡。

**上下文窗口**: 128K tokens（GPT-4o / o3-mini）
**定价**: API Token 按量计费

> 💡 Codex CLI 的一大特色是它有个"会话预算"功能，可以在达到 Token 上限前提醒用户，方便控制成本。

### 6️⃣ Hermes Agent — 全能型选手的代价

Hermes Agent 是 Nous Research 开发的通用 AI Agent，支持广泛的自定义工具和技能系统。

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~1,500 |
| 平台工具定义（terminal、file、web、memory 等 15+ 工具） | ~4,000-5,000 |
| MCP 工具扩展（按需加载） | ~1,000-3,000 |
| 技能 / Memory 注入 | ~500-2,000 |
| Kanban 上下文 | ~300-500 |
| **合计前置Token** | **~7,300-12,000** |

Hermes Agent 的前置 Token 波动范围非常大，原因在于它的**插件化架构**：

- **基础平台**：17 个默认 CLI 工具（terminal、file、web、code_execution、computer_use 等），每个工具都有完整的 JSON Schema 参数定义，这部分的 Token 消耗非常大。
- **技能系统**：每次对话会加载匹配的技能，每个技能的说明和指令都会追加到上下文中。
- **MCP 扩展**：若开启了 12306 候票、文件系统等 MCP Server，还要额外加载这些工具的定义。
- **Memory**：持久化记忆和用户画像数据也会在每轮对话注入。

> 好消息是：Hermes Agent 有**上下文压缩（Context Compression）** 机制，在对话变长时会压缩历史，保护上下文窗口不被撑爆。不幸的是，压缩生效前该消耗的 Token 已经消耗了。

**上下文窗口**: 随底层模型而定
**定价**: 完全免费开源，自备 API Key

---

### 7️⃣ Pi — 对话式AI的非典型选手

Pi 是 Inflection AI 推出的对话式 AI 助手，主打情感陪伴和自然对话，并非专门的编码工具。但它确实被不少开发者用作编码辅助。

| 项目 | 估算 Token 数 |
|------|-------------|
| 系统提示词 | ~1,200 |
| 角色/人格设定（Pi 以温暖体贴著称） | ~1,000 |
| 工具定义（极少，主要是对话工具） | ~300 |
| **合计前置Token** | **~2,500** |

Pi 的前置 Token 是最少的之一，因为它**不需要工具调用**。它不读文件、不执行代码、不操作终端 — 这些功能 Pi 根本没有。对于纯粹的**讨论型编码辅助**（比如"这段代码的逻辑是什么？"、"帮我设计一个 API 方案"），Pi 确实够用，而且前置开销极低。

但需要执行代码、调试程序、操作文件系统时，Pi 就完全无力了。

**上下文窗口**: 未公开（估计 ~32K）
**定价**: 免费（基础版），Inflection-2 模型

## 成本换算：前置Token到底值多少钱？

以 Claude Code 搭配 Claude 4 Sonnet 为例（当前最流行的编码方案）：

- **前置Token**: ~7,200 tokens
- **定价**: Claude 4 Sonnet 约 $15/百万输入tokens（按 Anthropic API 定价）
- **每次对话的前置成本**: 7,200 ÷ 1,000,000 × $15 = **$0.108**

也就是说，每次你开一个新对话窗口，在打字之前已经花掉了 **1 毛钱人民币**（约合 0.78 元人民币按 7.2 汇率）。

如果每天开 20 个新对话（重度使用者），仅前置 Token 每月就要花：
- 20 × 30 × $0.108 = **$64.8/月**

而 Copilot 补全模式的前置成本仅为：
- 600 ÷ 1,000,000 × $15 = **$0.009/次**，不到 1 分钱

**直观对比（按 Claude 4 Sonnet 定价计，¥7.2/USD）：**

| 工具 | 每次前置成本 | 每天 20 次月成本 |
|------|------------|----------------|
| Claude Code | ¥0.78 | ¥466 |
| Copilot 补全 | ¥0.05 | ¥32 |
| Hermes Agent | ¥0.79-¥1.30 | ¥474-¥778 |
| OpenCode | ¥0.38 | ¥227 |

> 当然，如果你用的是 API 定价更低的模型（比如 DeepSeek V4），前置成本会大幅下降。但 **Token 量本身还是在那里**，只是单价变了。

## 选型建议

### 按使用场景推荐

**如果你很在意 Token 成本**（自费 API 重度用户）
→ 首选 **Copilot 补全模式**（前置 ~600 tokens）或 **OpenCode**（~3,500 tokens）

**如果你需要最强 Agent 能力**（不介意前置开销）
→ **Claude Code**（生态成熟、文档丰富）或 **Hermes Agent**（扩展性强、支持 MCP/技能）

**如果你是 IDE 用户**（习惯了 VS Code）
→ **Cursor**（无缝集成、上下文感知好）或 **Copilot Chat**（性价比高）

**如果你仅需讨论代码**（不需要执行）
→ **Pi**（免费、前置极低、对话体验好）或 **Copilot Chat**

**如果你想要开源+可控**
→ **Hermes Agent**（可定制工具、技能系统强大）或 **OpenCode**（轻量、易上手）

### 省钱小技巧

1. **尽量复用对话**：不要频繁开新对话。一话题一对话，在同一个对话中连续提问可以分摊前置 Token 成本。
2. **用轻量模式处理简单任务**：改个变量名这种小事别开 Agent 模式，用补全模式就够。
3. **自建模型**：如果你有 GPU 资源，用本地模型跑 OpenCode 或 Hermes Agent，前置 Token 不花一分钱。
4. **精简工具链**：如果用 Hermes Agent，可以把不需要的工具和 MCP Server 禁掉，能省 2000-4000 tokens 的前置开销。
5. **关注提示词缓存**：Anthropic 等平台提供 Prompt Caching，相同系统提示词重复命中时大幅降价（约降价 90%）。

## 写在最后

前置 Token 是 AI 编码 Agent 的一笔**隐形税**。它不像模型输出的 Token 那样直观可见（你打字→它输出→扣费，一目了然），而是默默固定在每次请求的头部。工具越强大、能力越多，这笔税就越重。

但这并不意味着我们要一味追求低前置 Token。工具的价值在于**它能帮你完成什么**。一个前置 7,000 Token 但节省了你 30 分钟 debug 时间的 Agent，远比一个前置 500 Token 但帮不上忙的工具划算。

**先算账、再选型**，用数据做决策，才是理性开发者的做法。
